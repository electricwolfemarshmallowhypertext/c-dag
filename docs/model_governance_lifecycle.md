# Model Governance Lifecycle

This project is designed to fit a structured governance lifecycle.

## Stages

1. Design
2. Data review
3. CPD estimation
4. Expert review
5. Validation
6. Approval
7. Deployment
8. Monitoring
9. Incident review
10. Retirement

## Practical expectations

- Every stage should produce artifacts (docs, evidence, approvals).
- Promotions between stages should require explicit sign-off.
- Production incidents should feed back into design and validation updates.
